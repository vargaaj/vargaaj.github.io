Graduate School Projects

Android Programming

In this class, we studied Android APIs and learned to build significant Android applications. The course had a practical focus, with significant in-class programming, programming assignments and a large project. The course was taught entirely in Kotlin. While the course focuses on Android, we learned general principles of software engineering and mobile app development.

Project 1: Peck

Description: The idea for the game is that the user clicks a button and a sequence of English words appears in the area under the button. Also, each word appears individually (with a light grey background) in the game area, splayed out pretty randomly. You have to click each word in the sequence order in the time allotted. If you click a word out of order, the word flashes red. If you click all the words in the correct order, the game ends and you get points.

Project 2: Fling

Description: This project involves developing an interactive Android game with two modes: Jump and Fling. The Jump mode requires a puck to move between four predetermined positions upon user interaction, ensuring precise alignment with the game area's borders. The Fling mode introduces physics-based movement where the puck must be flung into a randomly selected green goal border within a countdown timer to score points. The puck follows realistic motion with friction and elastic collisions upon hitting non-goal borders. The application maintains state persistence when switching between modes and is built as a single-activity Android application.

Project 5: Barnes-Hut Using Message Passing Interface (MPI)

Description: In this project, we implemented an astrophysical N-body simulation using the Barnes-Hut algorithm in C++ with MPI (Message Passing Interface) to enable parallel computation. The Barnes-Hut algorithm is an efficient O(n log n) solution for N-body problems, leveraging a hierarchical tree structure to approximate gravitational forces between celestial bodies. Our implementation involved constructing a quadtree (for 2D simulation) to spatially partition the system and compute force interactions based on the Multipole Acceptance Criteria (MAC).

Key Skills:
  • Gained proficiency in distributing computation and managing communication across multiple processes using the Message Passing Interface in C++.
  • Implementation of quadtree structures for spatial partitioning and efficient force calculation.
