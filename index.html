Locksmith

Locksmith is a password management application that is used to store, update, and delete passwords and their associated information in a quick and secure manner. This app was built for my final project in my Android Programming graduate course. Please see the demo video here: Demo Video

Key Features:
  • Firebase Authentication for secure user login and registration
  • Firebase Firestore for real-time database management and data storage
  • Jetpack Compose for building a modern and responsive user interface
  • Kotlin Coroutines for asynchronous programming and background tasks
  • Compose Multiplatform for cross-platform development, allowing the app to run on both Android and iOS devices
  • Ktor for making network requests and handling API calls
  • End-to-end password encryption for secure storage of sensitive data

Android Programming

Project 4: Reddit

Integrated Reddit’s API to fetch dynamic content and focused on intuitive UI navigation and smooth user experience.

Project 3: Music Player

Description: Developed a fully functional music player app featuring essential playback controls including Play, Pause, Next, and Previous. Implemented Shuffle and Loop functionality for enhanced user experience.

Project 2: Fling

Description: This project involves developing an interactive Android game with two modes: Jump and Fling. The Jump mode requires a puck to move between four predetermined positions upon user interaction, ensuring precise alignment with the game area's borders. The Fling mode introduces physics-based movement where the puck must be flung into a randomly selected green goal border within a countdown timer to score points. The puck follows realistic motion with friction and elastic collisions upon hitting non-goal borders. The application maintains state persistence when switching between modes and is built as a single-activity Android application.

Project 1: Peck

Description: The idea for the game is that the user clicks a button and a sequence of English words appears in the area under the button. Also, each word appears individually (with a light grey background) in the game area, splayed out pretty randomly. You have to click each word in the sequence order in the time allotted. If you click a word out of order, the word flashes red. If you click all the words in the correct order, the game ends and you get points.

Parallel Systems

This class offered a comprehensive foundation in the principles of parallel systems, coupled with hands-on experience in a wide array of both classical and contemporary methods for managing and leveraging concurrency. It covered essential topics such as shared memory synchronization, the architecture of parallel systems like GPUs, and explored distributed parallel frameworks, providing a deep understanding of the techniques used to optimize performance in modern computing environments.

Project 1: Prefix Scan and Barriers

Description: This project aimed to deepen our understanding of parallel prefix scan algorithms and to explore their performance when using different synchronization primitives.

Key Skills:
  • C/C++
  • Utilized the Pthread library to implement multithreading, including the creation and joining of threads.
  • Employed synchronization techniques such as locks to manage concurrent access to shared resources.
  • Semaphore Barrier Implementation
Project 2: KMeans with CUDA

Description: Implemented the K-means clustering algorithm using CUDA and Thrust to gain hands-on experience with GPU programming. The project focused on parallelizing the algorithm, addressing GPU-specific challenges like memory management (global and shared memory), synchronization, and performance optimization.

Key Skills:
  • CUDA programming and parallelization
  • Thrust: C++ parallel algorithms library
  • Memory management and GPU optimization
  • Performance profiling and identifying bottlenecks
Project 3: BST Equivalence with GO

Description: The goal of this assignment was to write a program in a language that restricts the programming model to simplify dealing with concurrency: Google's Go. We used channels, go-routines, and signaling to compute binary search tree equivalence.

Key Skills:
  • Gained proficiency in Google’s Go language, particularly in the context of concurrency and parallelism
  • Used channel signaling and locks to synchronize concurrent operations and ensure correct program behavior
Project 4: Rust 2 Two-Phase Commit (2PC)

Description: The goal of this project was to gain exposure to using programming tools that leverage language-level type systems to manage concurrency and provide safety guarantees. We implemented 2PC across multiple processes communicating with each other using the inter-process communication offered by the ipc-channel crate.

Key Skills:
  • Gained foundational knowledge of implementing distributed protocols like the two-phase commit (2PC).
  • Learned how to effectively manage concurrency and safety using Rust's ownership model and type system.
  • Used the ipc-channel crates for communication between multiple processes, gaining understanding of its mechanisms and limitations.
  • Improved proficiency in Rust, including memory safety, type guarantees, and idiomatic use of the language's concurrency paradigm.
Project 5: Barnes-Hut Using Message Passing Interface (MPI)

Description: In this project, we implemented an astrophysical N-body simulation using the Barnes-Hut algorithm in C++ with MPI (Message Passing Interface) to enable parallel computation. The Barnes-Hut algorithm is an efficient O(n log n) solution for N-body problems, leveraging a hierarchical tree structure to approximate gravitational forces between celestial bodies. Our implementation involved constructing a quadtree (for 2D simulation) to spatially partition the system and compute force interactions based on the Multipole Acceptance Criteria (MAC).

Key Skills:
  • Gained proficiency in distributing computation and managing communication across multiple processes using the Message Passing Interface in C++.
  • Implementation of quadtree structures for spatial partitioning and efficient force calculation.
